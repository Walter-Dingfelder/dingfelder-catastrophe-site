Project overview

We are sourcing the field hardware for scene-level C.A.T.A.S.T.R.O.P.H.E. support.

These tablets are intended for high-consequence industrial and emergency-response environments where guidance, communications continuity, location awareness, and practical human handoff matter under pressure.

Production path 01

Android rugged tablet

Enterprise-managed Android device capable of kiosk/dedicated-device operation, repeatable provisioning, fleet control, and controlled update behavior.

Production path 02

Windows rugged tablet

Windows rugged device capable of kiosk/assigned-access operation, local admin restriction, controlled updates, encryption, and customer-IT-friendly management.

Important

Do not quote communications as add-ons

Wi-Fi, Bluetooth, GPS/GNSS, LTE/5G cellular, and SMS/text capability must be available on the proposed production tablet configuration.

Mandatory communication stack

Every proposed tablet must support five communication and location paths.

Vendors must clearly identify how each function is supported, including integrated radios, carrier compatibility, SIM/eSIM options, SMS path, and any required service or software layer.

Wi-Fi

Wi-Fi

Required for Starlink connection, VOIP, A.I.R.O.N. / V.A.U.L.T. sync, and approved network communications.

BT

Bluetooth

Required as an alternate short-range tablet-to-A.I.R.O.N. base connection when Wi-Fi is unavailable.

GPS

GPS / GNSS

Required for persistent location, tablet finder, responder support, Google Maps or facility-map projection, and last-known-location recovery.

5G

LTE / 5G Cellular

Required for emergency data fallback, offsite support, carrier-based fleet deployment, SMS support, and continuity if local infrastructure is degraded.

SMS

SMS / Text

Required for emergency alerts and fallback messaging. Vendor must state whether SMS is native cellular SMS, carrier-supported, or gateway-supported.

Hardware and operating requirements

Baseline device features requested.

Core hardware

  • Rugged tablet form factor suitable for industrial and response environments
  • Approximate display size: 10–12 inch class
  • Bright display suitable for field conditions
  • Touch operation suitable for real-world response use
  • Front and rear camera support
  • Reliable speaker and microphone for field use
  • Full-shift battery capability preferred
  • Docking and charging support required
  • Device serialization / asset tracking preferred

Security, provisioning, and updates

  • Kiosk / restricted-use mode required
  • Repeatable fleet provisioning required
  • Controlled update process required
  • No surprise production updates during emergency-use operation
  • Encryption and access restriction preferred where supported
  • Remote lock/wipe and MDM compatibility preferred
  • Documented recovery/re-provisioning path requested
Location and tablet finder

The tablet must frequently refresh and retain a Last Known Snapshot.

Persistent tablet location is a life-safety and operational-continuity requirement. The solution should support practical tablet finder workflows and projected location on Google Maps or a comparable facility/location layer.

Location

GPS/GNSS location with frequent refresh and last-known-position retention.

Status

Timestamp, battery state, device status, connection state, and active communication path.

Projection

Ability to feed persistent location to Google Maps or a facility-map view.

Recovery

Tablet finder support using last-reported location and timestamp after signal loss.
